- `docs/Svid_Download_Options_Dialog_PRD_v1.1.md`

# 1. Product Contract

The dialog must obey this user-facing contract:

| UI choice | Meaning | Engine behavior |
| --- | --- | --- |
| Recommended | Balanced default for most users. Avoid unnecessarily huge files and favor playback compatibility. | Use a bounded, compatible quality intent, currently up to 1080p unless global/app defaults say otherwise. |
| Best available | Highest downloadable quality available for this item. | Do not let H.264/MP4/FPS preferences lower the result. Prefer true max quality and let yt-dlp choose compatible streams. |
| Choose quality | User picks one specific target quality. | Do not silently upgrade to best. Exact match if available; otherwise nearest lower or explicit warning/fallback. |
| Save as default | Save current default intent for future downloads. | Persist File Type, Quality intent, and relevant global advanced/path defaults. |
| Remember for platform | Save current default intent for this source platform. | Persist platform-specific File Type, Quality intent, and relevant advanced overrides. Platform preference wins over global default. |

Storage contract:

| Storage scope | Stores | Must not store | Backward compatibility |
| --- | --- | --- | --- |
| Global default | Default file type, default quality intent, optional specific quality target, save path when chosen, advanced global defaults. | Platform-specific source preferences. | Existing `preferredQuality` remains readable; new fields should fallback from it when absent. |
| Platform preference | Platform, file type, quality intent, optional specific quality target, advanced overrides relevant to that platform. | Save path. | Existing preferences with only `qualityText` + `mediaType` must still resolve. |
| Current dialog submission | Full selected `Quality`, file type, quality intent, one-time save path override, advanced overrides. | Persistent settings unless Save as default / Remember for platform is checked. | Existing `selectedQualities` call sites keep working. |

Persisted specific targets must be portable across videos. Do not persist per-video `encryptedUrl`, raw yt-dlp `formatId`, raw gallery resource URL, or localized `qualityText` as the primary default/remember target.

| File Type | Persistable specific target | Current-dialog only target |
| --- | --- | --- |
| Video | `targetHeight` such as 1080/720/480, optional `targetFpsCap` or `targetFpsPreference`. | Raw video format ID, raw `encryptedUrl`, source-specific format note. |
| Audio | `outputFormat` such as mp3/m4a/opus/wav, optional `targetBitrateKbps`. | Raw audio stream format ID. |
| Subtitle | `languageCode`, `isAutoGenerated`, `subtitleFormat`. | Source display language name. |
| Image | `selectionMode: all` for persistent defaults. | Item index range, gallery resource URL, one-off item selection. |
| Technical stream | Not persistable as global/platform default. | One explicit raw stream selection for the current dialog submission. |

Legacy `preferredQuality` migration is lazy read-time mapping. Do not rewrite stored settings on app start.

| Legacy value | New default when new intent fields are absent |
| --- | --- |
| `auto` | File Type: Video, Quality Intent: Recommended |
| `best` | File Type: Video, Quality Intent: Best available |
| `p1080` | File Type: Video, Quality Intent: Specific, targetHeight: 1080 |
| `p720` | File Type: Video, Quality Intent: Specific, targetHeight: 720 |
| `p480` | File Type: Video, Quality Intent: Specific, targetHeight: 480 |
| `audioOnly` | File Type: Audio, Quality Intent: Recommended |

Engine contract:

| Intent | Format selector | Sort options | Output container/remux | Warning behavior |
| --- | --- | --- | --- | --- |
| Recommended | Bounded compatible selector, currently targets up to 1080p unless settings dictate lower. | Compatibility-oriented settings may apply. | Use selected/global compatible container. | Warn only when fallback changes visible quality materially. |
| Best available | Dedicated highest-quality selector with auto codec/audio/FPS. | Must not prefer H.264/MP4/AAC over higher source quality. | Avoid forced MP4/remux unless required for successful merge; prefer source-compatible merge output. | Warn if resulting container differs from global preference. |
| Specific quality | Exact target or bounded nearest-lower selector. | Respect selected target without unrestricted best fallback. | Use compatible container for selected stream pair. | Return a user-visible fallback note when exact target is unavailable. |

`Best available` means the highest quality available to the app in the current authenticated session, with current cookies/login state. It must not promise premium/member/private/locked formats that extraction did not expose. If the source requires login, cookies, age verification, membership, or premium entitlement, the dialog/download flow should show a recovery/auth warning instead of presenting the locked format as available.

Selector API contract to define before implementing selector changes:

```text
FormatSelectionRequest
  qualityIntent: recommended | bestAvailable | specific | technicalStream
  fileType: video | audio | image | subtitle
  target?: PortableQualityTarget
  videoCodecPreference
  audioCodecPreference
  containerFormatPreference
  fpsPreference
  fallbackPolicy: exactOnly | nearestLower | nearestWithWarning
  forceRemuxPreference

FormatSelectionResult
  formatSelector?: String
  sortOptions?: String
  videoFormat?: String
  audioFormat?: String
  forceRemux?: bool
  selectedTarget?: PortableQualityTarget
  warning?: FormatSelectionWarning

FormatSelectionWarning
  code: exactUnavailable | containerChanged | authRequired | formatUnavailable
  requestedLabel: String
  resolvedLabel?: String
  messageKey: String
```

Home and Browser flows must consume the same warning model. If the warning is known before submit, show it in the dialog/footer. If it is resolved at submit/download start, show one non-blocking snackbar after enqueue/start and avoid duplicate warnings.

Advanced controls are contextual:

| File Type | Advanced controls allowed |
| --- | --- |
| Video | Format, subtitles-with-video, metadata, SponsorBlock, time range, platform options. |
| Audio | Audio output-related controls, metadata/thumbnail where supported, time range if supported. Hide video codec/container/FPS if they are ignored. |
| Subtitle | Subtitle language, subtitle format, auto-generated captions. Hide video/audio formatting, SponsorBlock, metadata, and time range. |
| Image | Gallery/image-relevant controls only. Hide video/audio/subtitle/SponsorBlock controls. |

State preservation when switching File Type:

| State | Preserve? | Rule |
| --- | --- | --- |
| Save location | Yes | Always preserve until dialog closes. |
| Save as default / Remember / Apply all | Yes | These are submission-scope controls, not media-type controls. |
| Quality selection | No | Recompute default quality for the newly selected File Type. |
| Video codec/container/FPS | Preserve internally, ignore when not applicable | Do not lose user edits, but do not apply them to Audio/Image/Subtitle downloads. |
| Audio output intent | Preserve internally for Audio only | Do not apply audio export settings to Video unless explicitly part of the selected video flow. |
| Subtitle language/format | Preserve for Video subtitle-with-video and Subtitle-only | Hide/ignore for Audio/Image. |
| Time range | Preserve for Video and Audio | Hide/ignore for Subtitle/Image. |
| SponsorBlock | Preserve for YouTube Video/Audio only | Hide/ignore for Subtitle/Image and non-YouTube sources. |
| TikTok watermark removal | Preserve only for TikTok media downloads | Hide/ignore elsewhere. |

# 2. Architecture Decisions

- Keep `Quality` and the download engine mostly intact. Add an explicit dialog intent layer instead of turning the primary UI into raw stream selection.
- Keep `DownloadConfig.selectedQualities` for compatibility with current Home/Browser call sites, but add metadata that describes why the selected quality was chosen.
- Treat `Best available` as an intent, not just the current `Quality(encryptedUrl: ytdlp:best:mp4)` object.
- Treat raw streams as Advanced technical choices. They must not appear in the default primary `Choose quality` flow, but they must remain reachable to avoid functional regression.
- Use one scroll owner in the dialog body. Advanced content should expand into the parent scroll, not create a nested scroll area.
- Do not change Rust FFI/download engine code during this plan. If Dart selector changes are insufficient, stop and create a separate engine-change plan.
- Add or update a selector API contract before changing selector behavior. Tasks that touch `format_selector_service.dart` must implement against that contract rather than adding local helper paths.

# 6. Risks and Mitigations

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Persisting new quality intent conflicts with existing saved qualityText preferences. | Medium | Keep backward compatibility: if old preference lacks intent fields, resolve by qualityText/mediaType as today. |
| Best available may produce larger files than users expect. | Medium | Keep Recommended as default and visually distinguish Best available as a deliberate high-size choice. |
| Removing unrestricted fallback may increase "format unavailable" failures. | Medium | Use bounded nearest-lower fallback and user-visible warning instead of unbounded best fallback. |
| Contextual Advanced may hide a setting power users expect. | Low | Preserve hidden state and expose raw/technical options under a clearly labeled advanced diagnostics/stream section later. |
| Adding localization keys across five files may create missing translation drift. | Low | Update all existing translation JSON files in the same task. |
| Current working tree has partially implemented behavior. | High | Task 0 must reconcile the baseline before any source edits. |
| Best available requires engine changes beyond Dart selectors. | High | Stop and create a separate engine-change plan before touching Rust/FFI. |
| Persisted specific target stores source-specific identifiers. | High | Portable target schema forbids persisted `encryptedUrl`, raw format ID, resource URL, and localized quality text. |
